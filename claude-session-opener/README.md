# Claude Session Opener

Claude Pro/Max サブスクリプションの「5時間セッション制限」を、毎朝決まった時刻を
起点にリセットさせることを狙って、最小トークンのタスクを自動送信する
**試験的（experimental）** Home Assistant アドオンです。

## ⚠️ 目的と限界（必ず読んでください）

- Home Assistant の **Anthropic 統合**（`console.anthropic.com` の API キー、従量課金）と、
  Claude Pro/Max サブスクリプションの **5時間セッション** は完全に別システムです。
  API キー経由のリクエストはセッションの起点に一切影響しません。
  → このアドオンでは API キーは使用しません。
- 5時間セッションを起点にできるのは、Claude.ai / Claude Code に
  **サブスクリプション認証（OAuth ログイン）した状態での対話的利用のみ** です。
- 2026年6月15日付の課金変更により、`claude -p`（非対話実行）や Agent SDK 経由の
  呼び出しは対話利用の使用制限（5時間セッション）にはカウントされず、
  別枠の「Agent SDK 月次クレジット」から消費される仕様に変更されました
  （実施状況は変動の可能性があるため要確認）。
  → つまりこのアドオンを `claude -p` で組んでいる以上、**そもそも目的（5時間
    セッションの起点リセット）を達成できない可能性があります。**
- 上記の不確実性を踏まえ、このアドオンは試験的機能として実装しています。
  効果があったかどうかは、必ず [動作確認方法](#動作確認方法) に従って
  自分の目で確認してください。**効果が確認できなかった場合は
  [代替案](#効果が確認できなかった場合の代替案) を参照してください。**

## インストール

1. Home Assistant の **設定 → アドオン → アドオンストア** を開く
2. 右上のメニューから **リポジトリを追加** を選択し、以下の URL を入力する
   ```
   https://github.com/hgn32/ha-addons
   ```
3. **Claude Session Opener** を選択して **インストール**

## 初回セットアップ（サブスクリプション認証）

このアドオンは API キーを使わず、Claude.ai アカウントでの OAuth ログインで
認証します。**初回のみ手動での認証が必要です。**

### 方法1: アドオンのサイドバーパネルから（推奨）

Web Terminal や `docker exec` を使わず、アドオン単体でログインを完結できます。

1. アドオンを起動する
2. Home Assistant のサイドバーに表示される **「Claude Login」** パネルを開く
   （表示されない場合は、アドオンの **設定 → Ingress で表示** が有効になっているか確認してください）
3. **「ログインを開始」** ボタンを押すと、認証 URL が表示されます
4. その URL リンクを開き、自分のブラウザで Claude.ai の **サブスクリプションアカウント**
   でログインを完了する
5. ログイン完了後にサイトに表示される認証コードをコピーし、
   パネルの入力欄に貼り付けて **「送信」** を押す
6. 「✅ ログインに成功しました」と表示されれば完了です。認証情報は
   `/data/claude-credentials/` に保存され、コンテナを再起動しても維持されます。

コードが正しく認識されない場合は、コピーが不完全な可能性があります。表示された
案内に従って貼り直してください。うまくいかない場合は **「キャンセル」** を押して
最初からやり直せます。

### 方法2: ターミナルから（代替手段）

Web Terminal & SSH アドオン等が使える場合は、コンテナに入って直接コマンドを
実行することもできます。

```
docker exec -it addon_claude_session_opener /bin/bash
claude auth login --claudeai
```

（`--claudeai` は Claude サブスクリプションでのログインで、省略時のデフォルトです。
`--console` は API キー課金になるため **絶対に指定しないでください**。）
ログイン後、以下でサブスクリプション認証が有効になっているか確認できます。

```
claude auth status --text
```

### 共通の注意事項

**環境変数 `ANTHROPIC_API_KEY` は絶対に設定しないでください。** 設定されている
と API キー認証が優先され、従量課金になり本来の目的を達成できません。ログイン
パネルでは、ログイン済みの場合に認証方式（OAuth かどうか）も表示されるので、
API キー認証になっていないか確認できます。

## `schedule_time` オプションの設定

アドオンの **設定** タブから `schedule_time` を `HH:MM`（24時間表記）の形式で
設定します。デフォルトは `07:00` です。

```yaml
schedule_time: "07:00"
```

**`schedule_time` は常に日本時間（JST / Asia/Tokyo）として解釈されます。**
コンテナ内で `TZ=Asia/Tokyo` を明示的に固定しているため、Home Assistant
ホスト側のタイムゾーン設定（UTC など）には影響されません。デフォルトの
`07:00` はそのまま「日本時間の朝7時」を意味します。

毎日この時刻になると、最小トークンのタスク（`"ok"` という1メッセージ）を
Claude Code CLI 経由で送信します。

## 実行コマンドについて（技術メモ）

`ping.sh` は以下のコマンドを実行します。

```bash
claude -p "ok" --model haiku --output-format json --no-session-persistence
```

- **`--bare` は意図的に使用していません。** `--bare` を付けると Anthropic
  認証が `ANTHROPIC_API_KEY` / `apiKeyHelper` に固定され、OAuth
  （サブスクリプション）認証情報が一切読まれなくなります。これはこのアドオンの
  目的（従量課金 API キーを使わずサブスクリプションで起点にする）と正反対の
  挙動になるため、使用していません。
- **`--max-turns` は現行の Claude Code CLI に存在しないため使用していません。**
  代わりに1回きりのプロンプトなので `--max-turns` 相当の制御は不要です。
- `--no-session-persistence` は `--print`（`-p`）専用のオプションで、
  セッションをディスクに保存せず再開もできないようにします。

## 動作確認方法

このアドオンが実際に5時間セッションを起点にできているかは、**必ず
Claude Code 内の `/usage` コマンドで確認してください。**

1. アドオンのログ（**設定 → アドオン → Claude Session Opener → ログ**）で、
   スケジュール時刻に `ping.sh` が実行され、エラーなく完了していることを確認する
2. 実行直後に、自分の端末で `claude` を起動し `/usage` を実行して、
   5時間セッションのカウンターが起動時刻（`schedule_time`）を起点に
   リセットされているかを確認する
3. 数日間にわたって確認し、安定して効果があるかを判断する

`/data/session_opener.log` にも実行結果とあわせて、この呼び出しが
セッション枠にカウントされたかどうかは `/usage` で検証する必要がある旨の
注記を毎回記録しています。

## 効果が確認できなかった場合の代替案

上記の課金変更（2026年6月15日）により `claude -p` がセッション枠に
カウントされない場合、このアドオンでは目的を達成できません。その場合、
**現状もっとも確実な方法は、人間が朝7時に手動で Claude.ai（または
Claude Code）を開いて一言送ることです。** 自動化に固執せず、確認が取れる
までは手動運用を併用してください。

## データ・ログのパス

| 種類 | パス |
|---|---|
| 認証情報 | `/data/claude-credentials/`（コンテナ再起動後も維持） |
| 実行ログ | `/data/session_opener.log` および HA のアドオンログ |
| アドオン設定 | HA の **設定 → アドオン → Claude Session Opener → 設定** タブ |
| ログイン画面 | サイドバーの **「Claude Login」** パネル（Ingress 経由） |

ログには API キーや認証トークンは一切出力しません。
