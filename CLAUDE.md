# ha-addons プロジェクトルール

## HAアドオンのバージョンアップ手順

上流（Suwayomi等）の新しいバージョンがリリースされたときは、以下を必ず行うこと。

1. **`config.json` のバージョンを上げる**
   - `"version"` フィールドをインクリメントする（例: `"0.12"` → `"0.13"`）

2. **`CHANGELOG.md` を更新する**
   - アドオンディレクトリ内の `CHANGELOG.md` に新バージョンのエントリを追記する
   - 形式:
     ```
     ## <新バージョン>

     - <変更内容>

     ## <旧バージョン>
     ...
     ```
   - このファイルはHome AssistantのUI上でアドオンの更新履歴として表示される

## ユーザーへの案内・機能設計の原則

- ユーザーは基本的に Home Assistant の画面（アドオンのログタブ・設定タブ・
  Ingress パネル等）以外にアクセス手段を持たない前提で考えること。
  `docker exec`、SSH、コンテナ内のファイルを直接開く、といった操作を
  ユーザーに要求する案内や実装を作らない（例:「/data/xxx.log を見てください」
  とだけ言って終わらせない）。
- ログや状態など、ユーザーに確認してほしい情報は、HA のログタブに出力するか、
  アドオン自身の Ingress パネル等、HA の画面内で完結する手段で見られるように
  実装すること。ファイルパスを案内するだけで「見る手段」を実装しないのは禁止。
