## 0.22

- Suwayomi Server を v2.3.2238 にアップデート(上流 Hotfix の取り込み)

## 0.21

- `webUI` を永続化・バックアップ対象に戻した
  - Suwayomi 本体は起動のたびに webUI ディレクトリを削除→実ディレクトリとして
    作り直して静的アセットを展開し直すため、0.20 の symlink による対象外化は
    起動時に実体へ置き換わって機能していなかった(実際のバックアップには
    webUI の実データが入っていた)
  - 機能していなかった symlink 化コードを削除し、README も実態に合わせて修正

## 0.20

- `webUI`(サーバーが自動再取得する静的アセット)と `backups`(.tachibk)も
  バックアップ・永続化の対象外にした(ユーザー要望)
  - `webUI`: 実害なし。本体が起動時に自動で再取得する
  - `backups`: **これまでの「更新前にバックアップ作成→更新後に復元」という
    復旧手順が変わります。** サーバー上に置いたバックアップファイルは
    アドオン更新のたびに消えるようになったため、更新前に Suwayomi WebUI から
    バックアップファイルを一旦 **PC にダウンロード**しておき、更新後に
    その**ファイルをアップロードして復元**してください
    (サーバー上のフォルダを選択する形の復元は更新を跨いでは使えません)
  - `extensions` は要望を確認のうえ、影響が大きい(更新のたびに再インストール
    が必要になる)ため**従来どおり永続化を維持**
  - 副作用として、ホストの `/addon_configs/<slug>/backups` から直接
    `.tachibk` ファイルを見る・取り出すことはできなくなった。閲覧は
    Suwayomi WebUI または本アドオンの Summary viewer(ingress パネル)から
    行うこと

## 0.19

- `bin` / `cache` / `logs` / `downloads` / `thumbnails` を永続化・バックアップの
  対象外にした(コンテナ内への symlink になり、HA バックアップには実体が入らない)
  - ダウンロード済みチャプターは従来(〜0.16)どおり「再起動では残る・
    アドオン更新で消える」挙動に戻る
- H2 マイグレーションが残す退避コピー(`database.mv.db.*.backup`、DB と同サイズ)を
  起動時に削除するようにした(バックアップ肥大の防止)

## 0.18

- 修正: `/config` 配下に root 所有のファイル・ディレクトリが混ざっていると
  拡張機能アイコンの保存などが `Permission denied` になる問題
  (例: `extensions/icon/*.tmp`)。起動時に所有権を自動修復するようにした。
  HA のバックアップ復元後に権限がずれた場合も次回起動時に自動で直る

## 0.17

- **修正: アップデートのたびにライブラリ・既読状態・設定が数日前に巻き戻る問題**
  - 原因: ファイル単位のシンボリックリンクが、Suwayomi 本体の H2 マイグレーションや
    上流起動スクリプトの `sed -i` によって実ファイルに置き換えられ、以後のデータが
    コンテナ内(アップデートで消える領域)にしか書かれなくなっていた
  - 対策: Tachidesk データディレクトリ全体を `/config/tachidesk` に移して永続化。
    既存データは初回起動時に自動移行される
  - これによりダウンロード済みチャプターや各種設定もアップデートを跨いで保持される
- **重要(復旧手順)**: 前回アップデート以降のデータはコンテナ内にあったため、
  このアップデートの適用時点で失われる。適用「前」に Suwayomi の
  WebUI → 設定 → バックアップ で手動バックアップを作成しておき、
  適用「後」に WebUI から復元すると最新状態に戻せる
  (自動バックアップ(.tachibk)は `/addon_configs/<slug>/backups` に残っている)
