## 0.14

- 【高速化】合成の CPU スレッド数(`--cpu_num_threads`)を**ホストの物理コア数に自動設定**するようにした
  - 従来は `6` 固定だった。実測では合成速度はスレッド数が物理コア数と一致するとき最速で、
    超えるとむしろ遅くなる(コンテキストスイッチ増)。4 コアのホストでの実測では
    6 固定は最適値(4)より約 1.5 倍遅かった
  - 既定 `cpu_num_threads: 0` で**物理コア数**に自動設定する。SMT/HT の論理コア(`nproc`)
    まで使うと onnxruntime 推論はむしろ遅くなるため、`/proc/cpuinfo` から物理コア数を
    求める(例: Ryzen 5 3500U は 4 コア 8 スレッド → 論理 8 ではなく物理 4 を採用)。
    物理コア数を取得できない環境では `nproc` にフォールバック
  - 設定で 1 以上を指定すると手動で固定できる(通常は物理コア数以下を推奨)
- 【メモ】GPU 以外で動作を軽くするには、上記スレッド数の最適化に加え、使うスタイルだけを
  読み込む `load_all_models: false` + `preload_speaker_ids` の併用が有効(メモリと起動時間を削減)

## 0.13

- 【追加】エンジン設定(CORS Policy Mode / Allow Origin)とユーザー辞書・プリセットを
  `/data` に永続化。アドオンを再起動しても設定・辞書が消えなくなった
  - `XDG_DATA_HOME=/data/xdg` を指定するだけで、設定ファイル・ユーザー辞書・プリセットが
    すべて `/data/xdg/voicevox-engine/` に保存される(実測確認済み)
- 【変更】固定で渡していた `--cors_policy_mode all` を撤去
  - この引数は設定ファイルより優先されるため、付けたままだと設定ページで CORS を
    変更しても無視され保存が効かなかった。撤去してエンジン既定(localapps)に変更
  - CORS はブラウザが別オリジンからエンジン API を叩く場合のみ関係する。TTS
    (ha-voicevox-tts はサーバー間通信)・設定ページや `/docs` の表示には影響しないため、
    既定 localapps でも通常の利用に支障はない(実測確認済み)
  - 変更が必要な場合は設定ページ(`http://<HAのアドレス>:50021/setting`)で all 等に
    切り替えると `/data` に保存され、再起動後も保持される

## 0.11

- 【診断】起動監視ログを追加。原因切り分けに必要な情報をすべてログタブに出力します
  - 起動時に `options.json` の内容と実際の起動コマンド(CORS 設定含む)を表示
  - ポート 50021 で待ち受けを開始するまで、15 秒ごとに経過時間・実メモリ(RSS)・仮想メモリ(VSZ)・設定上限を表示
  - 待ち受け開始時に「エンジン起動完了(所要秒数・エンジン版)」を表示。この行が出るまで 50021 は Connection refused になります
  - エンジンプロセスが終了した場合は必ず exit code をログに記録(これまで無言で止まっていたケースを可視化)
- 【修正】`max_memory_mb` の既定値を `0`(無制限)に変更。0 より大きい値を設定した場合のみ `ulimit -v` を適用(上限も 16384 に拡大)
  - 0.10 イメージの実測で、エンジンは起動完了までに仮想メモリ(VSZ)を **約 8.1GB** 要求することを確認(実メモリ RSS は約 2.4GB)。このため 0.10 の既定 2048MB では `memory allocation failed` で abort、8000MB でもわずかに不足して起動不能だった
  - 制限を設定する場合は 12288 以上を推奨(実測ピーク 8044MB + 余裕)
- 【追加】`load_all_models` オプション(既定 `true`=従来どおり)。`false` にすると全モデルの事前読み込みを行わず数秒で待ち受けを開始するため、モデル読み込みフェーズの問題かどうかを 1 操作で切り分けできます
- 【追加】Supervisor watchdog を設定(`/version` を定期ヘルスチェック)。エンジンが待ち受けに到達しない・応答しなくなった場合に Supervisor が異常を検知して自動再起動します(アドオン情報画面の「番犬」トグルで有効化)
- 【追加】`preload_speaker_ids` オプション。`load_all_models: false` と組み合わせると、指定したスタイル ID のモデルだけを読み込みます(実測: 全モデル読み込み RSS 約2.4GB/VSZ 約8.0GB → スタイル1件のみ RSS 約0.24GB/VSZ 約1.3GB)。未指定のスタイルも初回利用時に自動読み込みされるため使用可能です
